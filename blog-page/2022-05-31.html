Hacking Journey


Hi, my name is Falk and this is my attempt on documenting a hacking journey.


Where to start?

There is so much to learn, this[1] great talk by David Brumley recommends to start with ctf's.

Ultimately I decided to do "Basic" on hackthissite.org[2].

Note: I will posting all answers that I got on here


The "Idiot" test

The description reads as following

This level is what we call "The Idiot Test", if you can't complete it, don't give up on learning all you can, but, don't go begging to someone else for the answer, thats one way to get you hated/made fun of. Enter the password and you can continue.

It's trivial, you just have to open the dev tools in your browser and look for the html comment that contains the password

Other Tasks

Here is a quick rundown of the other tasks and my way of solving them up to basic8, because as of writing this I haven't completed more tasks.


basic2

Here you just have to push submit, because Security Sam forgot to upload the password file which holds the password that is then compared to what the user entered.

basic3

Here you can find a filepath in the html (you have to select the password input with the browser dev tools) to a file named password.php, navigate to that file, and you get the password.

basic4

The password is sent to Secsam per email, just edit the email address in the html so that the password is sent to you instead to sam.

basic5

Same as in basic4

basic6

I had to figure out how the cipher works, that was the first time I used an ascii-table.

basic7

I actually had to google this one, because I didn't know the cal command, turns out that that isn't important. In the end you just had to execute two commands simultaneously and then go to the file which contains the password
